# 🔌 Open WebUI 集成指南

> 让 Open WebUI 拥有一个真正的“长期记忆大脑”。

这个目录是将 **六层记忆系统 (Memory System)** 接入 **Open WebUI** 的完整套件。  
接入后，你的 AI 助手将不再是一个“聊完即忘”的模型，而是一个能**自动回忆、自动反思、持续成长**的个人伙伴。

---

## 🏗️ 交互原理图

一图胜千言，这就是它的工作流：

```text
      用户提问 (User Query)
            │
            ▼
    ┌─────────────────┐
    │  Inlet Filter   │ ⚡️ 提问前拦截
    └───────┬─────────┘
            │ 1. 🔍 意图识别 (路由决策：该查简历？还是查图谱？)
            │ 2. 🧠 调用 Memory API (按需并行检索 L1-L5)
            │ 3. 💉 注入 Context (把召回的记忆塞给模型)
            ▼
    ┌─────────────────┐
    │    LLM Model    │ 🤖 生成回答
    └───────┬─────────┘
            │
            ▼
    ┌─────────────────┐
    │  Outlet Filter  │ 💾 回答后拦截
    └───────┬─────────┘
            │ 1. 📝 提取对话内容
            │ 2. 🤖 调用 Memory API (自动分类)
            │ 3. 📦 沉淀为长期记忆 (L1/L2/L3/L4/L5)
            ▼
      用户收到回答 (Final Response)
```

## 🧠 核心架构：为什么它比普通记忆更强？

大多数记忆插件只做了一件事：把对话存进向量库。  
而本系统构建了完整的**六层认知架构**，针对不同类型的信息使用不同的存储和检索策略。

### 1. 六层分治设计 (L0-L5)

| 层级 | 名称 | 存什么？ | 存在哪？ | 解决了什么问题？ |
| :--- | :--- | :--- | :--- | :--- |
| **L0** | **Profile** (画像) | 身份、性格、偏好、指令 | YAML | 解决“你是谁、我是谁”的稳定性问题，防止人设漂移。 |
| **L1** | **Semantic** (知识) | 技能、方法论、通用知识 | Qdrant (向量) | 解决“经验复用”问题，不被时间线淹没的长期沉淀。 |
| **L2** | **Episodic** (事件) | 日报、会议、具体经历 | Qdrant (向量+时间) | 解决“我做过什么”的事实追溯，带时间戳和来源。 |
| **L3** | **Relational** (关系) | 人物、团队、项目依赖 | **Neo4j (图谱)** | 解决“谁和谁有关、谁是关键节点”的复杂推理问题。 |
| **L4** | **Reflective** (反思) | 周报、复盘、成长评价 | Qdrant (向量) | 解决“光有经历没有成长”的问题，沉淀高阶认知。 |
| **L5** | **Aspiration** (目标) | 计划、OKR、里程碑 | YAML + 向量 | 解决“下一步做什么”的导向问题，不仅仅是回忆过去。 |

### 2. 智能路由机制 (Router)

系统**不会**笨拙地对每个问题都去查所有层，而是根据 **Inlet Filter** 的意图识别结果，精准调用：

- **问“那个项目谁负责”** 👉 激活 **L3 (关系图谱)** + L1 (知识)，精准定位人物关系。
- **问“上周做了什么”** 👉 激活 **L2 (事件)**，按时间倒序召回。
- **问“我的优势是什么”** 👉 激活 **L4 (反思)** + L1 (知识)，合成自我评价。
- **问“下一步学什么”** 👉 激活 **L5 (目标)**，对比当前进度与目标差距。

> **价值**：这种机制极大地减少了上下文噪音（Context Noise），让 AI 的回答更精准、更专业，而不是胡乱堆砌检索结果。

### 3. 硬核技术栈

这不是玩具，而是按照企业级标准搭建的本地数据平台：

- **向量数据库**: `Qdrant` (高性能 Rust 向量引擎)
- **图数据库**: `Neo4j` (处理复杂实体关系)
- **API 框架**: `FastAPI` (高并发异步接口)
- **推理引擎**: `Ollama` (本地 LLM + Embedding)

## ⚡️ 功能展示：自动意图识别 (Copilot)

Inlet Filter 内置了强大的意图识别，无需手动指令，直接对话即可触发高级能力：

### 1. 📝 自动生成简历 (`/synthesize/resume`)
- **触发词**：`简历`、`CV`、`履历`、`述职`
- **底层逻辑**：聚合 **L1技能 + L2项目 + L3影响力 + L5目标**，按 STAR 法则合成。

### 2. 📊 工作状态雷达 (`/synthesize/work_status`)
- **触发词**：`最近进展`、`工作情况`、`处理到哪`
- **底层逻辑**：扫描 **L2事件**，识别阻塞点和里程碑，生成状态报告。

### 3. 🪞 自我画像合成 (`/synthesize/self_profile`)
- **触发词**：`我是谁`、`我的画像`、`你怎么看我`
- **底层逻辑**：综合 **L0设定** 和 **L4历史反思**，生成客观的 AI 视角评价。

### 4. 🚨 项目风险预警 (`/synthesize/project_radar`)
- **触发词**：`风险`、`阻塞`、`卡点`、`延期`
- **底层逻辑**：专查负面反馈和未完成的 L2/L5 记录。

### 5. ⚖️ 决策副驾驶 (`/synthesize/decision_copilot`)
- **触发词**：`怎么决策`、`方案对比`、`该选哪个`
- **底层逻辑**：调取过往类似经验 (L1) 和价值观 (L0) 辅助 Trade-off。

### 6. 🗺️ 学习引擎规划 (`/synthesize/learning_engine`)
- **触发词**：`学什么`、`学习计划`、`复盘计划`
- **底层逻辑**：基于 **L5目标** 差距，推荐接下来的学习路径。

## 📦 组件清单

本目录包含三个核心文件，按需取用：

### 1. `memory_inlet_filter.py` (🌟 推荐必装)
- **角色**：记忆召回器 & 路由控制器
- **作用**：实现上述所有的自动检索、分层调用和意图路由。

### 2. `memory_outlet_filter.py` (🌟 推荐必装)
- **角色**：记忆存储器
- **作用**：聊天结束后，自动提取高价值信息，调用后端 `/remember` 接口，让系统自动分类存储。

### 3. `memory_tools.py` (🛠️ 可选)
- **角色**：手动工具箱
- **作用**：给模型 Function Call 能力，适合需要手动干预记忆（如“修正这条记忆”、“强制查询某天日志”）的场景。

## 🚀 三分钟快速接入 (Open WebUI)

### 第一步：确认后端存活 ✅

确保你的 Memory System 后端已经启动（参考根目录 README）。

```bash
curl http://localhost:8000/health
# 预期返回: {"status": "healthy", ...}
```

### 第二步：配置 Inlet Filter (自动召回) 📥

1. 打开 Open WebUI **管理员面板** -> **Functions** (函数)。
2. 点击 **+** 号创建新函数。
3. 填入名称：`Memory Inlet`。
4. 将 `memory_inlet_filter.py` 的内容完整粘贴进去。
5. 开启 **Inlet** 开关。
6. 点击保存，并在你的模型设置里启用这个 Filter。

### 第三步：配置 Outlet Filter (自动存储) 📤

1. 同样在 **Functions** 面板创建新函数。
2. 填入名称：`Memory Outlet`。
3. 将 `memory_outlet_filter.py` 的内容完整粘贴进去。
4. 开启 **Outlet** 开关。
5. 保存并启用。

> 🎉 **恭喜！** 现在你的 Open WebUI 已经拥有长期记忆了。试着问一句：“帮我生成一份简历草稿。”

## ⚙️ 进阶配置 (环境变量)

你可以在 Open WebUI 的函数配置界面（Valves 区域）或环境变量中设置这些参数：

| 变量名 | 默认值 | 说明 |
| :--- | :--- | :--- |
| `MEMORY_API_BASE` | `http://host.docker.internal:8000` | 记忆系统后端的地址 |
| `MEMORY_API_KEY` | (空) | 如果后端开启了鉴权，请在此填入 Key |
| `MEMORY_GLOBAL_SOURCES` | (空) | 全局召回白名单（如 `import_worklog,manual`），让新会话也能读到这些旧数据 |

### 💡 关于会话隔离 (Scope)

默认情况下，我们开启了**会话隔离**：
- **写入时**：数据标记为 `chat::{current_chat_id}`。
- **读取时**：只读 `当前会话数据` + `全局白名单数据`。

**为什么要这么做？**
防止你在 A 窗口聊“写科幻小说”，结果在 B 窗口聊“工作周报”时，AI 突然产生幻觉说“你要发射飞船”。

## 🛠️ 故障排查

如果接好后没反应，请检查：

1. **URL 通不通**：Open WebUI 容器能访问到 `http://host.docker.internal:8000` 吗？
2. **Key 对不对**：后端 `.env` 里的 `MEMORY_API_KEY` 和 Filter 里填的一样吗？
3. **Filter 开没开**：在聊天界面的模型设置里，是否勾选了这两个 Filter？
4. **日志看一眼**：查看后端日志 `data/system/classification_log.jsonl`，有没有新的 `recall` 或 `remember` 记录？
